HTTP redirect check
HTTP redirect detection
After DNS resolves the domain to an IP, the web server at that IP may send an HTTP redirect response (301 Moved Permanently, 302 Found, etc.) telling the browser to load a different URL instead. Common examples: http:// → https://, abc.com → www.abc.com, or old domain → new domain. This check attempts to follow HTTP redirects — if CORS blocks the request, the server doesn't allow cross-origin checks from this page, but the redirect may still exist when accessed directly in a browser.
